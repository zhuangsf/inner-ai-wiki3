### 问题: 文件管理器在有大图片的地方滑动慢，甚至出现OutOfMemory的处理
解决方案: 文件管理器中,在快速查看进入图片,或者浏览存有大图片的目录时,有图片文件列表的地方,都需要有图片文件的缩略图,而缩略图是由图片文件解码出来的.已经解码出来的图片的缩略图会有一个缓存来存储,而第一次加载的过程中,都有一个图片解码的过程,而且是同时在解码多张图片. 图片来自外部,可能是各式各样的大小、分辨率等,在解码图片的如果遇到特别大的图片,解码会耗费大量的内存,而手机的内存资源有限,有时候解码一张图片资源就能耗尽所有内存,出现OutOfMemory或者出现用户点击无反应. 解决这问题,在解码图片的时候,对图片文件的大小和分辨率做限制,超出支持的范围,就不再解码,而用缺省的图片资源替换. 所以,客户也可以根据自己剩余可用内存的情况调节图片文件的大小和分辨率限制的大小,在文件src/com/sprd/fileexplorer/util/FileUtil.java readBitmap()方法中.

### 问题: 点阵字体和矢量字体的差别是什么？
解决方案: 1、点阵字体 点阵字体也叫位图字体,其中每个字形都以一组二维像素信息表示.点阵字体通过点阵表现字形,其本质上就是一组图片. 如果图片上的点非黑即白,就称之为黑白点阵;如果图片上的点有不同的亮度甚至颜色,则称之为AA点阵. 优点:显示速度快. 缺点:不可缩放,字体显示不平滑. 2、矢量字体 矢量字体通过数学曲线来表述,它包含了字形边界上的关键点,连线的导数信息等,字体的渲染引擎通过读取这些数学矢量, 然后进行一定的数学运算来进行渲染. 优点:可缩放,字体显示平滑. 缺点:因矢量显示需要进行运算,所以显示速度相对点阵比较慢.

### 问题: 手机在使用过程中为什么会出现啸叫问题？
解决方案: 手机在使用状态下啸叫的原因: 总的来说是一种闭环的自激现象,也就是说在手机使用时,从喇叭发出的信号经过一定的衰减之后翻过来又送回到MIC,当回授的信号大于原先送入的信号时,这时音频回路的总的增益大于1时,就产生了啸叫,形成啸叫的途径大约又三种: (1)喇叭发出的声音经过空间从机壳的外面回授到MIC (2)喇叭发出的声音经过机壳的内部的声音通道或空间回授到MIC (3)另一个途径是因为喇叭和MIC是装在同一个机壳上的,如果喇叭或MIC的减震效果不好的话,那么喇叭的振动,通过机壳传到MIC 另外MIC的前端如果有空腔的话,会对某一高频产生共振,从而产生高频啸叫.

解决方案: 展讯平台上slog中android log包括 main,system,radio,events,kernel. 在手机shell终端输入如下命令: $ adb shell slogctl enable //开启android log $ adb shell slogctl disable //关闭android log 设置完成之后,可以使用adb shell slogctl query 确认当前配置是否已经更改. 如: stream kernel on 0 7 stream radio on 0 0 stream system on 0 8 stream events on 0 0 stream main on 0 7 第三个参数即红色字体为on表示已经打开,为off表示关闭.
