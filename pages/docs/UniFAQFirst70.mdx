### 问题: 接收灵敏度的定义及计算公式
解决方案: 定义:接收机能够接收到的并且还能正常工作的最低电平强度. 接收灵敏度跟很多东西有关,如噪声系数,信号带宽,解调信噪比等,灵敏度一般来说越高(数值越低),说明其接收微弱信号的能力越强,但也带来容易被干扰的毛病,对于接收机来说,灵敏度只要能满足要求即可. 公式为: -174 + NF +10lgB + SNR ( NF为噪声系数,B为信号带宽,单位为Hz. SNR为解调信噪比 ,单位为dB) 例如:解调带宽为10KHz, 噪场系数取10dB,解调信噪比为12dB,则可以算得接收机灵敏度为-112dBm .

解决方案: 展锐平台Andorid9.0及之后的版本不支持应用分身功能,原因如下: 1.功能涉及到对framework的大量改动,可能会引入一些未知问题,对系统稳定性产生不确定性. 2.AppClone功能依赖于多用户功能,go版本等多用户功能关闭的版本对应无法支持此功能. 3.应用分身功能本身涉及到的适配工作量巨大. 基于以上原因,平台9.0及之后默认不支持应用分身功能,同时不建议您基于平台 source code 开发应用分身功能.

### 问题: Android10.0 平台 GSI 升级规则
解决方案: Android 10.0 平台 GSI与安全补丁匹配规则引入了新的变化,Google 升级要求是 GSI的SPL >= 安全补丁的SPL. 举例说明: 如果当前安全补丁的SPL :2020-04-05, 则可选的GSI 的SPL:2020-04-05,2020-05-01,2020-05-05 等, 不可选的GSI的SPL:2020-04-01,2020-03-05等, 故建议在合入安全补丁前,需优先升级GSI,如果升级顺序不符,存在GSI版本无法开机的风险. 无法开机原因说明: 若 GSI的SPL比安全补丁旧,在first init阶段会发生GSI Rollback,导致GSI版本在lock状态下无法开机. 具体代码参考: system/core/init/first_stage_mount.cpp IsStandaloneImageRollback方法中触发GSI rollback.
