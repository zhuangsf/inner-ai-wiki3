Tee的全称为Trusted Execution Environment，即可信执行环境。
相对有个概念叫REE，全称为Rich Execution Environment。
对于android手机系统，ree即指android运行环境，而tee则是指tos运行环境。
一般应用逻辑都是跑在android运行环境的，对于一些安全性要求高的逻辑可以拆分到tee运行环境之下执行，比如加密运算，生物识别算法（如指纹/人脸算法）。那些运行在tee中的逻辑也是以一个一个应用的形式运行，这些应用就叫ta（Trusted APP），比如指纹算法应用就叫指纹ta。

平台指纹/人脸都是tee方案的，特别是人脸方案是平台自研开发的，不涉及第三方，所有代码及库文件（包括ta）都已经包含在IDH中，不需要额外提供。
